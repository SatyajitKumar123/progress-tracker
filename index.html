Week 01
Django — Core Foundations
Django Project Structure — What Every File Does and Why Create fresh project. Explain settings.py, urls.py, wsgi.py, manage.py out loud to yourself. Write file tree on whiteboard.
Django Models — Fields, Relationships, Migrations OneToOne, ForeignKey, ManyToMany. Write 3 different model sets. Run migrations each time. Draw ER diagram on whiteboard.
Django Migrations Deep Dive — What Happens Underneath makemigrations vs migrate. Inspect migration files. Fake migrations. Squash. Write the flow on whiteboard before touching keyboard.
Django ORM — CRUD 10 Different Ways create(), save(), bulk_create(). get(), filter(), exclude(). update(), delete(). Write same query 3 different ways each. No copy-paste.
Django ORM — filter, exclude, annotate, aggregate Count(), Sum(), Avg(), Max(). Q objects. F expressions. select_related vs prefetch_related. Apply each to TradeMind models.
Review Day Repeat Week 1 From Memory — Whiteboard Only No notes. No docs. Write everything you remember. Gaps = what you revise. This is the most important day of the week.
Week 02
Django — Views, URLs, Templates, Forms
Django Views — FBV vs CBV, When to Use Which Build same view both ways. Draw request → view → response on whiteboard. Write CBV inheritance chain from memory.
Django URLs — include(), Namespacing, Dynamic Paths path() vs re_path(). App namespaces. Reverse URLs. Build multi-app URL config from scratch 3 times.
Django Templates — Context, Tags, Filters {% for %}, {% if %}, {% block %}, {% extends %}. Custom template tags. Pass TradeMind data to a template page.
Django Forms — ModelForm, Validation, save() clean() methods. Custom validators. is_valid() flow. Build a trade entry form for TradeMind.
Django Django Admin — Register, Customize, list_display list_display, list_filter, search_fields, readonly_fields. Register all TradeMind models. Add custom actions.
Review Day Repeat Week 2 From Memory — Whiteboard Only Whiteboard: FBV vs CBV diagram, URL flow, template inheritance tree. Code one complete view-url-template chain from scratch.
Week 03
Django — Advanced + Auth + Build Day
Django Signals — pre_save, post_save, When to Use vs Avoid Connect signals. Disconnect. Avoid signal hell. Add a TradeMind signal that logs whenever a trade is saved.
Django Middleware — What It Is, Write One Custom Middleware order matters. Write a request logging middleware for TradeMind. Draw the middleware stack on whiteboard.
Django Settings — Dev vs Prod, Environment Variables Split settings files. python-decouple or django-environ. Never hardcode secrets. Apply to TradeMind.
Django Django Auth — User Model, Login, Logout, Permissions AbstractUser vs AbstractBaseUser. Custom user model. Groups and permissions. Add user-scoped data to TradeMind.
Linux Linux Week — Navigation + Permissions + Processes ls/cd/mkdir/rm/cp/mv until automatic. chmod/chown. ps/kill/top. Run Django via terminal only this week. No GUI.
Build Day 🔨 Build One Complete Django App From Scratch — No Help Pick any idea. Models, views, urls, templates, auth, admin. Complete in one day. This proves Week 1-3 landed. This is your milestone.
Week 04
DRF — Serializers, Views, Auth
DRF Serializers — ModelSerializer, Validation, create/update validate_field(). validate(). SerializerMethodField. Write TradeMind serializers 3x from scratch. No copy-paste.
DRF APIView — GET, POST, PUT, DELETE From Scratch request.data, request.method. Response(). Status codes. Build full CRUD for TradeMind trades endpoint.
DRF ViewSets + Routers — Write Same API Both Ways ModelViewSet vs APIView. When each makes sense. DefaultRouter. Build TradeMind API both ways, feel the difference.
DRF Permissions — IsAuthenticated, IsOwner, Custom Write 2 custom permissions. Object-level permissions. Apply ownership check to TradeMind so users only see their trades.
DRF Authentication — SessionAuth, TokenAuth, JWT SimpleJWT — access + refresh tokens. Token rotation. Add JWT to TradeMind. Test with Postman or HTTPie.
Review Day Repeat All 5 DRF Topics From Memory Whiteboard: DRF request lifecycle diagram. Then build serializer + APIView + JWT auth from scratch, no notes.
Week 05
DRF Advanced + Git Mastery
DRF Nested Serializers — Read vs Write, SerializerMethodField Writable nested. source parameter. Depth option. Build trade with nested portfolio data in TradeMind API.
DRF Filtering — django-filter, SearchFilter, OrderingFilter FilterSet. Multiple filters combined. Add filtering to TradeMind — filter trades by symbol, date range, status.
DRF Pagination — PageNumber, LimitOffset, Cursor Custom page sizes. Override get_paginated_response(). Add pagination to TradeMind trade list endpoint.
Git Git — init, add, commit, log, diff, branch, merge Write every command by hand first on whiteboard. Create deliberate merge conflicts. Resolve them. Rebase vs merge — draw the difference.
Git Git — Remote, Push, Pull, .gitignore, Professional README Connect TradeMind to GitHub properly. Write a real README: what it does, why, tech stack, how to run. Pin it. This is your portfolio now.
Build Day 🔨 Rebuild TradeMind API Completely From Scratch — No Old Code Full DRF API. Auth, permissions, filtering, pagination. One day. If you can build it from memory, you can explain it in any interview.
